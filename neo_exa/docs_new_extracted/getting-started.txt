Getting Started with OxiTerm
OxiTerm is a server-side rendered TUI framework. You write declarative THTML markup; the framework renders it to a terminal grid and streams ANSI diffs over SSH or WebSocket. Clients need only a terminal.

1. Installation
Install Rust: https://rustup.rs
Clone and build:
git clone https://github.com/yourorg/oxiterm &amp;&amp; cd oxiterm
cargo build --release
Binary: target/release/oxiterm

2. First Application
Create hello.thtml:
&lt;box style=&quot;flex-direction: column; width: 80; height: 24; bg: #0f172a;&quot;&gt;
  &lt;box style=&quot;height: 3; bg: #1e293b; align-items: center;
              padding-left: 2; border-style: single; border-color: #334155;&quot;&gt;
    &lt;text style=&quot;fg: #38bdf8; height: 1;&quot;&gt;Hello, OxiTerm!&lt;/text&gt;
  &lt;/box&gt;
  &lt;box style=&quot;padding: 2;&quot;&gt;
    &lt;text style=&quot;fg: #e2e8f0; height: 1;&quot;&gt;Welcome to server-side TUI.&lt;/text&gt;
  &lt;/box&gt;
&lt;/box&gt;

Start server (dev mode, no auth):
oxiterm serve hello.thtml --port 2222 --no-auth

Connect from any terminal:
ssh localhost -p 2222

3. Hot Reload
OxiTerm watches the served file. Edit and save — all connected sessions update immediately without reconnecting.

7. Input Fields
&lt;input bind-value=&quot;query&quot; placeholder=&quot;Search...&quot;
       style=&quot;height:1;border-style:single;border-color:#38bdf8;&quot;/&gt;
&lt;text bind-state=&quot;query&quot; style=&quot;fg:#94a3b8;height:1;&quot;/&gt;

Tab to focus → type → Enter commits to state[&quot;query&quot;]
Backspace removes last character
event-htmx fires after commit if set

11. App Server
For custom logic connect an external HTTP server:
OXITERM_APP_SERVER=http://localhost:3000/events oxiterm serve app.thtml

OxiTerm POSTs on every event-htmx action:
{ &quot;action&quot;: &quot;save_form&quot;, &quot;state&quot;: { &quot;email&quot;: &quot;...&quot; }, &quot;session_id&quot;: 42 }
