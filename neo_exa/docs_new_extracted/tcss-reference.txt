TCSS Reference
TCSS (Terminal CSS) is OxiTerm&apos;s stylesheet language. It uses standard CSS syntax (selector { property: value; }) with a property vocabulary tuned for the character grid.

1. Cascade and Specificity

Priority
Source
Example
1 (lowest)
Tag selector
box { border-style: single; }
2
Class selector
.card { padding: 1; }
3
ID selector
#header { bg: #1e293b; }
4 (highest)
Inline style attribute
style=&quot;fg: #38bdf8; height: 1;&quot;

💡 Last rule wins for same-selector conflicts. No specificity calculation within one layer.

2. Style Block
&lt;style&gt;
  text  { fg: #e2e8f0; }
  .panel { border-style: rounded; padding: 1; flex-direction: column; }
  .active { border-color: #38bdf8; }
  #sidebar { width: 24; bg: #1e293b; }
&lt;/style&gt;

7. Inline Styles
The style=&quot;&quot; attribute accepts semicolon-separated TCSS declarations in any order. Trailing semicolon is optional.
&lt;box style=&quot;flex-direction: column; bg: #0f172a; padding: 1; height: 10;&quot;&gt;
&lt;text style=&quot;fg: #38bdf8; height: 1;&quot;&gt;Hello&lt;/text&gt;

8. Properties NOT Supported
font-size, font-weight, font-family (monospace only, no bold/italic)
display: none/block (use bind-show)
position: absolute/fixed (no absolute positioning)
overflow: scroll/hidden (document scroll via PgUp/PgDn only)
gap (use margin between children)
flex: N shorthand (flex-grow not yet implemented)
opacity, transform, transition, animation
