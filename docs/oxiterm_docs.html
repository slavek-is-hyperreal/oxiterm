OxiTerm — Mapa Architektoniczna

OxiTerm to platforma TUI (Terminal User Interface) działająca w modelu SSR (Server-Side Rendering). Serwer parsuje pliki THTML, oblicza layout, renderuje do bufora komórek, tworzy ANSI diff i przesyła przez SSH lub WebSocket. Klient nie potrzebuje żadnej instalacji.

🔑 Kluczowe właściwości

  • SSR — rendering całkowicie po stronie serwera
  • THTML/TCSS — markup jak HTML/CSS, ale dla terminala
  • Auto-negocjacja: Kitty → Sixel → Unicode half-blocks
  • Backpressure: BoundedFrameChannel (drop oldest)
  • Hot Reload: inotify → InputEvent::Reload
  • Predictive Echo: lokalne echo przed potwierdzeniem
  • BSU/ESU: synchronized update (brak tearingu)
  • A11y: LinearFrameSink + DBus/AT-SPI
  • Metryki Prometheus na :9090
  • Zewnętrzny App Server: fire-and-forget POST JSON

Przepływ Danych — Pełny Cykl

1. Inicjalizacja połączenia SSH

2. Główna pętla renderowania (EventLoop::run)

3. Przepływ input → state → render

Klikięcie przycisku (event-htmx)

1
Terminal → ReactorThread
Raw bytes SSH → sanitize_frame() → InputStateMachine.feed_slice() → InputEvent::KeyPress lub MouseEvent
2
EventLoop odbiera event
recv_timeout() zwraca event, last_activity reset. Dla mouse: HitTester.find_node(col, row) → NodeId
3

Na znalezionej node: czytamy attrs.event_htmx, np. "inc:count". Wywołujemy StateManager.apply_action()
4

StateManager.get_dirty_nodes() zwraca subskrybentów. doc.mark_dirty(). Ustawia needs_render = true
5
Re-render → diff → send
Pełny pipeline: layout → renderer → diff. Tylko zmienione komórki generują ANSI commands. Owija BSU/ESU.
6

Fire-and-forget POST do zewnętrznego serwera. Odpowiedź JSON → apply_state_patch() → InputEvent::StatePatched

Pipeline Renderowania — wg Terminala Docelowego

OxiTerm auto-negocjuje zdolności terminala przez DA1 query (ESC[c) i Kitty Graphics probe (ESC_Gi=31...). Wybiera najlepszy backend graficzny.

Kitty Graphics Protocol — pełna grafika pikselowa

✓ Najwyższa jakość. Obsługiwane przez: kitty, WezTerm (częściowo), Ghostty.

Szczegóły protokołu Kitty (APC sequence)

ESC_G a=T, f=100, c={cols}, r={rows}, m={more}; {base64_chunk} ESC\
     ^       ^       ^         ^         ^
     |       |       |         |         0=last chunk, 1=more
     |       |       |         ilość wierszy terminala
     |       |       ilość kolumn terminala
     |       f=100: PNG, f=32: raw RGBA 
     a=T: Transmit+Display (a=q: query, a=d: delete)

Każdy chunk ma maks. 4096 bajtów base64. Duże obrazy są dzielone na wiele fragmentów (m=1), ostatni m=0.

Sixel Graphics — terminale Sixel-compatible

⚠ Obsługiwane przez: iTerm2, mlterm, xterm +sixel, VTE 0.67+. Niższa jakość koloru niż Kitty.

Unicode Half-blocks — fallback dla każdego terminala

⚠ Najmniejsza jakość graficzna. Działa wszędzie. Używane gdy brak Kitty i Sixel.

Mechanizm

Każda komórka terminala = 2 "piksele" pionowo. Górny piksel = ▀ (U+2580 UPPER HALF BLOCK), dolny = ▄ (U+2584). Kolor fg to kolor górnego piksela, bg to kolor dolnego.

Każda cell terminala (1 kol × 1 wiersz) → 2 piksele:
  ┌──────────────┐
  │ ▀ fg=color_1 │  ← pixel (x, y*2)
  │   bg=color_2 │  ← pixel (x, y*2+1)  
  └──────────────┘
Pełne bloki: ▀ ▄ █ ░ ▒ ▓ + box-drawing chars: ─ │ ┌ ┐ └ ┘ ╭ ╮ ╰ ╯

OxiTerm używa PUA-B (Private Use Area B) do stabilizacji layoutu na różnych emulatorach — znaki z zakresu U+100000+ mają gwarantowaną szerokość 1 komórki, unikając problemów z dwuszerokimi znakami CJK.

Limity

Brak wsparcia dla obrazów, SVG, Lottie, Rive ani video. Renderer w renderer.rs sprawdza profile.supports_kitty_gfx i profile.supports_sixel — jeśli oba false, pomija rasteryzację całkowicie i renderuje tylko komórki tekstowe + half-blocks dla tła.

Web — xterm.js + WASM Canvas

✓ Działa w każdej przeglądarce bez instalacji. Frontend: xterm.js. Renderer: oxiterm-web WASM.

Accessibility — Screen Reader / DBus

Aktywowane przez oxiterm serve --a11y lub a11y_mode: true w config.

LinearFrameSink

Zamiast bufora 2D ANSI, dokument jest renderowany jako linearna sekwencja tekstowa. build_a11y_tree(doc) buduje drzewo A11yNode z rolami AT-SPI (Button, TextInput, Label, Container, itd.).

THTMLDocument → build_a11y_tree() → Vec<A11yNode>
   Screen  (role: Application)
   └── Box (role: Panel)
       ├── Text "Counter: 5" (role: Label)
       └── Button "Increment" (role: PushButton)

DBusBridge

DBusBridge::new() + open_tunnel() — otwiera proxy do lokalnego socket DBus. Wysyła zdarzenia AT-SPI2 (focus, text-changed, action) bezpośrednio do syntezatora mowy (Orca, Braille display).

Zidentyfikowane Błędy i Problemy

Błędy oznaczone komentarzami BUG-* w kodzie plus nowe wykryte podczas analizy.

IDPlikStatusOpisFix
BUG-M01 session.rs:ResizeDebouncer FIXED ResizeDebouncer.poll() powinien mierzyć czas od push (pushed_at), nie od last_update. Bez fixa: debounce nie działał po pierwszym resize. Kod używa self.pushed_at.elapsed() — poprawnie.
BUG-M02 document.rs:compact() FIXED Arena compact() musi zaktualizować root i dirty_nodes przez remap. Bez tego: NodeId wskazują na złe węzły po kompakcji. document.compact() aktualizuje root i dirty_nodes przez HashMap remap.
BUG-M03 parser/thtml.rs:sanitize_htmx_value FIXED Bez sanityzacji event-htmx: możliwe wstrzyknięcie escape sequences lub kontrolnych znaków w akcjach. sanitize_htmx_value() filtruje control chars i niebezpieczne znaki.
BUG-H01 session.rs:PredictiveEcho FIXED PredictiveEcho::confirm() musi usuwać pierwszy znak (FIFO), nie ostatni. Stary kod usuwał cały bufor przy mismatch. Implementacja FIFO z Vec chars i remove(0) przy match.
BUG-H06 / RATELIMIT-01 ssh/server.rs, ratelimit.rs FIXED Duplikacja sprawdzenia rate limit: raz w TCP accept (mod.rs), raz w channel_open_session. Podwójne sprawdzenie prowadzi do inconsystencji. Usunięto duplikat z channel_open_session. Jedyne sprawdzenie przy TCP accept.
BUG-BIND-SHOW-INCREMENTAL-01 layout/engine.rs:compute() FIXED Taffy cache powoduje, że bind-show nie przelicza gdy state zmienia widoczność węzła. Incremental update nie wykrywa zmiany Display::None. compute() zawsze wywołuje reset_nodes() (full rebuild). Wolniejsze, ale poprawne.
BUG-COMPACT-01 layout/engine.rs:reset_nodes() FIXED Po document.compact() node_map zawiera stare OxiNodeId → TaffyNodeId. Nowe NodeId po remap nie mają wpisów. Layout fails. reset_nodes() czyści TaffyTree i node_map (full rebuild przy następnym compute).
BUG-KITTY-GHOST-01 session.rs:AnsiFrameSink::send_frame() FIXED Przy przejściu z ramki z grafiką Kitty do ramki bez grafiki: stare image placement pozostają na ekranie jako "duch". Wykryto: front.graphics.is_empty() && back.graphics → delete_all_placements() przed diff.
BUG-SPINLOCK-01 backpressure.rs:blocking_recv() FIXED Stara implementacja: sleep(5ms) w pętli = busy-wait + dodatkowe 5ms latency. Zużywała CPU na idle sesje. parking_lot::Condvar.wait() — blokuje bez CPU burn, wakeup natychmiastowy przez try_send notify.
BUG-RESIZE-01 session.rs:EventLoop::run() FIXED DoubleBuffer resize przy każdym InputEvent::Resize powodował podwójną alokację. Debouncer teraz triggeruje jedną realokację. resize_debouncer.poll() w pętli render, nie przy odbiorze eventu.
BUG-NEW-01 session.rs:SessionRegistry::new() OPEN max_sessions hardkodowane na 20. OxiTermConfig::session.max_sessions (domyślnie 100) jest ignorowane. SessionRegistry::new() nie przyjmuje konfiguracji. W produkcji pozwala na max 20 sesji bez względu na config. Dodać parametr max_sessions: usize do SessionRegistry::new() i przekazać z config.
BUG-NEW-02 style.rs:parse_da1_response() OPEN Kitty APC i DA1 w if-else. Gdy terminal wysyła Kitty APC response (starts_with ESC_G) — ustawia tylko supports_kitty_gfx, pomija cały blok parsowania DA1 (?64, SGR mouse, Sixel). Jeśli terminal łączy obie odpowiedzi w jednym buforze, SGR mouse może nie zostać wykryty. Rozdzielić detekcję Kitty APC i DA1 na niezależne sprawdzenia (bez else).
BUG-NEW-03 render/renderer.rs:VIRTUAL_FS PARTIAL thread_local! VIRTUAL_FS. HashMap jest thread-lokalny. W async runtime Tokio, render może być wykonywany na różnych worker threads. Jeśli VIRTUAL_FS jest populowany na jednym wątku (np. przy ładowaniu assets), inny wątek go nie widzi. Dla non-WASM: używa std::fs::read() (brak problemu). Dla WASM: single-threaded. Problem potencjalny przy multi-thread populate + render. Rozważyć OnceLock globalny cache zamiast thread_local.
BUG-NEW-04 render/diff.rs:diff() wide chars PARTIAL Pozycja kursora po wide char na końcu wiersza. Gdy wide char (CJK, width=2) jest na ostatniej lub przedostatniej kolumnie: cur_x = x + 2, ale sprawdzenie if cur_x >= next.width resetuje do (0, y+1). Przy kolejnym innym znaku, MoveCursor do (0, y+1) jest pomijany jeśli cur_x/y = 0/y+1 (fałszywy cache hit). Rozważyć czyszczenie cur_x/cur_y na None po każdym wrap, nie ustawianie na (0,y+1).
BUG-NEW-05 session.rs:EventLoop weather poll PARTIAL try_recv() busy-poll na weather_rx. W każdej iteracji pętli (co 5ms gdy brak animacji) wywoływane jest weather_rx.try_recv(). To nie blokuje, ale przy braku danych weather, generuje ciągłe "miss" calls. Akceptowalne przy 5ms tiku, ale nieoptymalne. Można użyć mpsc::channel z timeout zamiast try_recv, lub flagi pending_weather_update.
Podsumowanie: 9 z 10 zidentyfikowanych bugów jest naprawionych (FIXED). BUG-NEW-01 (max_sessions) jest kluczowy do poprawki przed produkcją. BUG-NEW-02 może powodować brak myszy na niektórych terminalach.

Akcje HTMX (event-htmx)

event-htmx="inc:count"           → StateManager.apply_action("inc:count")
event-htmx="dec:count"           → counter - 1
event-htmx="toggle:show_panel"   → bool flip
event-htmx="set:tab=info"        → StateValue::Str("info")
event-htmx="append:items=new"    → StateValue::List push "new"
event-htmx="clear:input_text"    → reset do domyślnej wartości

Łańcuchy akcji (separator ; lub ,):
event-htmx="inc:count;set:status=active"

Nawigacja do innego widoku:
event-htmx="about.thtml"         → ładuje about.thtml jako nowy dokument

Warunek bind-show

<box bind-show="tab=info">...</box>     → widoczne gdy state["tab"] == "info"
<box bind-show="!logged_in">...</box>   → widoczne gdy state["logged_in"] == false
<box bind-show="count=0">...</box>      → widoczne gdy state["count"] == "0"

Ewaluacja: StateManager::evaluate_bind_show() przez StateEvaluator trait.
LayoutEngine ustawia Display::None dla False → węzeł nie zajmuje miejsca.
